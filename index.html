GUARDIAN RUNTIME

WHAT GUARDIAN STOPS

01
PII Detection
Scans every prompt and response for sensitive personal data before it moves. Native India DPDP Act support — Aadhaar, PAN, UPI IDs. Plus SSN, credit cards, passports, emails, phone numbers.
DPDP Act GDPR HIPAA CCPA
02
Secret Detection
Two-tier confidence engine catches exposed API keys with high accuracy. HIGH tier: known prefixes (sk-proj-, AKIA, ghp_). MEDIUM tier: generic KEY=value patterns. Never leaks credentials to an LLM.
OpenAI AWS GitHub Stripe Razorpay
03
Jailbreak Detection
Pattern-matched detection for DAN variants, instruction overrides, role-play injections, encoding tricks, and system prompt extraction attempts. No ML model required — zero latency.
DAN Injection Role-play Encoding
04
Input Optimizer
Automatically trims redundant tokens from prompts before they hit the LLM. Reduces cost without changing semantics. Tracks savings per session with estimated USD saved.
Token Reduction Cost Savings
05
FinOps & Cost Tracking
Real-time token counting and cost estimation per session. Per-agent spend limits enforced at the policy level. Full cost report per agent. Works across OpenAI, Anthropic, and Gemini pricing.
Token Counting Spend Limits Cost Reports
06
Multi-Provider Support
One SDK for all major LLMs. Swap providers in the YAML policy — zero code changes. Full provider abstraction means your security layer travels with your stack.
OpenAI Anthropic Gemini
07
YAML Policy Engine
Declarative, per-agent rules. Enable or disable each guard independently. Set spend limits, token caps, and provider defaults. Define it once in a YAML file — Guardian enforces everywhere.
Per-Agent Rules Declarative
08
Local Logging & Proxy
All violation logs stored at ~/.guardian/logs/. A built-in proxy server lets you intercept traffic from any HTTP client transparently. CLI tools: guardian status, guardian logs, guardian init.
Local Logs HTTP Proxy CLI

QUICKSTART

01
Install
Zero external dependencies for core detection. Optional extras for proxy and dashboard.
pip install guardian-runtime
02
Wrap your LLM call
Drop Guardian between your app and any LLM. One import, one object, fully governed calls.
from guardian_runtime import GuardianRuntime

gr = GuardianRuntime()

# Your normal LLM call — now governed
response = gr.complete(
  model="gpt-4o",
  messages=[{"role": "user", "content": user_input}]
)

# response.blocked → True if threat detected
# response.violations → list of what was caught
# response.estimated_cost_usd → spend this call
03
Configure your policy (optional)
Declarative YAML. Per-agent rules. Enable only what you need.
version: "1.0"
agents:
  default:
    input_guard:
      pii_detection: true
      secret_detection: true
      jailbreak_detection: true
    output_guard:
      pii_detection: true
      hallucination_check: false
    cost:
      max_input_tokens: 4000
      session_budget_usd: 1.00
04
Or use the scan API directly
For simpler use-cases — scan text without the full engine.
from guardian_runtime import scan_pii, scan_secrets

result = scan_pii("My Aadhaar is 9876 5432 1098")
# result.blocked → True
# result.type → AADHAAR
# result.severity → HIGH

result = scan_secrets("key = sk-proj-abc123...")
# result.blocked → True
# result.type → OPENAI_KEY
