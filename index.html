Why Guardian?

Features

Everything your AI stack
needs to stay safe

Production-grade guardrails, cost controls, and governance — all in one Python package.

🛡️
One line to govern any LLM call
Replace your raw API call with guardian.complete() and get automatic PII blocking, token compression, and audit logging.
1from guardian import Guardian
2
3guardian = Guardian.from_policy("policies/gemini.yaml")
4
5response = guardian.complete(
6  messages=[{"role": "user", "content": "What is my Aadhaar?"}]
7)
8
9if response.blocked:
10  print("Blocked:", response.violations[0].type) # → pii
🔬
Scan without any API key
Standalone PII and secret scanning that runs entirely offline — no model, no key, no latency.
1from guardian import scan_pii, scan_secrets
2
3scan_pii("Aadhaar: 2345 6789 0123").blocked # True
4scan_secrets("sk-proj-xxxxx").blocked # True
−40%
Token cost reduction
The built-in optimizer trims whitespace, deduplicates context, and prunes old history before your prompt hits the model.
🔒
10+ PII entity types
Aadhaar, PAN, UPI, SSN, credit cards, email, phone, passport, and more — fully regex-powered, zero cloud calls.
Aadhaar PAN UPI SSN Credit Card Passport Email Phone
🧱
40+ jailbreak patterns
DAN prompts, role-play overrides, encoding tricks, and goal-hijacking — Guardian catches them all before they reach the model.
DAN Role-play Encoding Override Goal-hijack
📋
Policy-as-Code via YAML
Define per-agent rules, token budgets, guard modes (block / redact / flag), and provider routing — all in a single YAML file you own.
1version: "1.0"
2agents:
3  default:
4    llm: { provider: gemini, default_model: gemini-2.0-flash }
5    input_guard: { pii_detection: true, jailbreak_detection: true }
6    output_guard: { pii_detection: true }
7    optimizer: { enabled: true, max_history_messages: 10 }
8    cost: { max_input_tokens: 8000 }

Compliance-ready from day one
Built as an assistive tool (not legal advice) to help teams operating under GDPR, HIPAA, CCPA, and India's DPDP Act. All audit logs remain local at ~/.guardian/logs/.

Quickstart
CLI
# Core
pip install guardian-runtime

# With PDF/DOCX converter + dev tools
pip install "guardian-runtime[optimizer,dev]"

# With Presidio NER-based PII
pip install "guardian-runtime[presidio]"
# 1. Set key
export GEMINI_API_KEY=your_key

# 2. Run
python examples/quickstart.py
# Initialise (optional license)
guardian init --key gdn_free_xxxxx

# Validate policy file
guardian validate policies/gemini.yaml

# See usage this month
guardian status

# Tail audit logs
guardian logs --tail 20

Privacy by Architecture

We can't read
your prompts.
By design.

Guardian makes exactly two types of network requests. That's it. Every word of your prompts stays on your machine.

✅
Your LLM provider
Calls go directly from your machine to OpenAI / Gemini / Anthropic via your own API key.
✅
guardian-ai.dev (license only)
Once per day: { key_hash, check_count } — never prompts, never violations.
We store: hashed key · plan type · usage count · payment records. That's all. No prompts. No responses. Ever.
