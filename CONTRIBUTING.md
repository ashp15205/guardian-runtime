# Contributing to Guardian Runtime

First off, thank you for considering contributing to Guardian Runtime! We appreciate all contributions, whether it's fixing bugs, improving documentation, or adding new features.

## Getting Started

1. **Fork the repository** on GitHub.
2. **Clone your fork** locally.
3. **Install dependencies** using `pip install -e ".[all,dev]"`.

## Development Workflow

- Create a new branch for your feature or bugfix.
- Ensure your code follows the existing style guidelines. We use `ruff` for linting.
- Add unit tests for any new features or bug fixes.
- Run tests using `pytest` to ensure everything works correctly.

## Submitting a Pull Request

1. Push your changes to your fork.
2. Open a Pull Request against the `main` branch of the upstream repository.
3. Provide a clear description of the changes you've made and the problem they solve.
4. Ensure CI passes successfully on your PR.

## Code of Conduct

Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms.

Thank you for contributing!
